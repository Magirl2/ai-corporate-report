# 제품 기획 및 우선순위 관리 (Product Planning)

## 목적
제품의 목표를 명확히 하고 가용 리소스를 효율적으로 배분하여 사용자에게 최고의 가치를 제공하기 위함입니다.

## 언제 사용하나
- 새로운 기능을 기획하거나 요구사항을 수집할 때
- 버그 수정과 기능 개발 사이의 우선순위를 조정할 때
- 태스크를 정의하고 완료 기준을 수립할 때

## 관련 파일
- `README.md`
- `.agent/agents/product-manager.md`
- `src/pages/DataNotice.jsx`, `PrivacyPolicy.jsx`, `Terms.jsx`

## 핵심 규칙
- 해결하고자 하는 사용자의 문제를 명확히 정의한다.
- 요구사항을 P0(필수), P1(중요), P2(보조) 등 우선순위에 따라 분류한다.
- 모든 작업은 실행 가능한 태스크 단위로 쪼개고 구체적인 완료 기준을 정한다.
- 보안, 법적 이슈, 핵심 사용자 경험(로그인/보고서)을 최우선으로 고려한다.
- 작업의 목적, 대상 파일, 테스트 방법을 포함하는 명세서를 작성한다.
- 사용자 체감 품질을 기준으로 성공 지표를 설정한다.

## 검증 방법
- 각 태스크에 목적, 대상 파일, 완료 기준, 테스트 방법이 포함되어 있는지 확인
- 현재 진행 중인 작업이 최상위 우선순위(P0)에 부합하는지 확인
- 법적 고지 및 데이터 신뢰도 요구사항이 실제 UI에 반영되었는지 확인

## 흔한 실수
- 한 번에 너무 큰 규모의 작업을 시도하여 진행 상황을 파악하기 어렵게 만드는 경우
- 기술적인 재미에 매몰되어 사용자에게 실질적인 가치가 낮은 기능에 집중하는 경우
- 작업의 완료 기준이 모호하여 최종 결과물의 품질이 들쭉날쭉한 경우
