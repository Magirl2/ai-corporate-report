# 인증 및 세션 관리 (Auth & Session)

## 목적
사용자의 로그인, 회원가입 및 세션 상태를 안전하게 관리하고 원활한 사용자 경험을 제공하기 위함입니다.

## 언제 사용하나
- 로그인/회원가입 로직을 수정할 때
- 사용자 세션 유지 및 권한 확인 기능이 필요할 때
- JWT 발급 및 검증 로직을 변경할 때

## 관련 파일
- `api/auth/login.js`, `signup.js`, `me.js`, `logout.js`
- `api/_lib/env.js`, `api/_lib/db.js`
- `src/pages/Login.jsx`, `Signup.jsx`
- `src/App.jsx`

## 핵심 규칙
- 비밀번호는 반드시 `bcrypt`를 사용하여 해싱 후 저장/검증한다.
- 세션 관리는 `httpOnly` 쿠키를 사용하여 XSS 공격으로부터 보호한다.
- 로그인 전 `/api/auth/me` 호출 시 401 반환은 정상적인 동작으로 처리한다.
- 로그인 실패(401)와 서버 설정 오류(500)를 명확히 구분하여 응답한다.
- API 응답 메시지를 프론트엔드에서 사용자에게 안전하게 표시한다.
- 세션 만료 시 적절한 로그아웃 처리를 수행한다.

## 검증 방법
- 실제 회원가입 및 로그인 흐름 테스트
- 잘못된 비밀번호 입력 시 401 에러 반환 확인
- 로그인 전후 `/api/auth/me` 응답 상태 확인

## 흔한 실수
- 비밀번호를 평문으로 저장하거나 검증하는 경우
- 쿠키 설정을 잘못하여 클라이언트에서 세션 정보를 읽지 못하는 경우
- 로그아웃 시 쿠키를 제대로 만료시키지 않는 경우
