# 법적 고지 및 데이터 신뢰성 (Legal & Trust)

## 목적
법적 리스크를 관리하고 데이터의 출처와 한계를 명확히 고지하여 사용자로부터 신뢰받는 서비스를 제공하기 위함입니다.

## 언제 사용하나
- 이용약관, 개인정보처리방침 등 정책 문서를 수정할 때
- 데이터 고지사항이나 투자 유의사항을 UI에 반영할 때
- AI 생성 결과물의 한계점을 안내해야 할 때

## 관련 파일
- `src/pages/DataNotice.jsx`, `PrivacyPolicy.jsx`, `Terms.jsx`
- `src/components/layout/Footer.jsx`
- `src/pages/SingleReportView.jsx`

## 핵심 규칙
- 데이터 출처 및 수집 방법에 대한 명확한 고지사항을 유지한다.
- "투자 권유가 아니며, AI 생성 결과물은 오류가 있을 수 있음"을 명시한다.
- 개인정보 항목, 보유 기간, 제3자 제공 여부 등을 정책에 정확히 반영한다.
- 사용자가 원문 데이터를 확인할 수 있도록 링크나 경로를 제공한다.
- 데이터의 최신성 한계(캐시 등)를 인지할 수 있도록 정보를 제공한다.
- 법률, 세무, 회계 등 전문 자문이 아님을 분명히 밝힌다.
- 모든 정책 페이지는 푸터(Footer)를 통해 언제 어디서든 접근 가능해야 한다.

## 검증 방법
- 푸터의 각 링크가 올바른 법적 고지 페이지로 연결되는지 확인
- 모바일에서 긴 약관 텍스트가 읽기 쉬운 레이아웃으로 표시되는지 확인
- 모든 페이지에 TODO나 Placeholder 문구가 남아있는지 확인

## 흔한 실수
- 타 서비스의 약관을 그대로 복사하여 회사명이나 서비스명이 틀린 경우
- AI의 답변이 너무 단정적이어서 투자 유의사항 고지를 무색하게 만드는 경우
- 개인정보 처리방침의 연락처나 이메일이 유효하지 않은 경우
