# DART 및 FMP 데이터 연동 (DART & FMP Data)

## 목적
신뢰도 높은 기업 재무 데이터를 수집하고, 국내외 기업 데이터 소스를 효율적으로 분기 처리하기 위함입니다.

## 언제 사용하나
- 새로운 재무 지표를 추가하거나 수집 로직을 변경할 때
- DART 공시 데이터 파싱 로직을 수정할 때
- 국내외 기업 구분 및 데이터 통합 로직이 필요할 때

## 관련 파일
- `api/_lib/dart-utils.js`
- `api/_lib/handlers/data/dart.js`, `dart-finance.js`
- `api/_lib/orchestrator.js`
- `api/_lib/sourceQuality.js`

## 핵심 규칙
- OpenDART API의 `corp_code`와 종목코드(`stock_code`)의 차이를 명확히 인지하고 사용한다.
- 기업명으로 `corp_code`를 찾는 Resolve 과정을 거친다.
- 국내 기업은 DART를 우선하며, 데이터 부재 시 FMP를 폴백으로 사용한다.
- 재무 지표(성장률, 이익률, ROE 등) 계산 시 0으로 나누기 에러를 방지한다.
- 재무 수치 단위(원, 달러 등)를 사용자에게 보여주기 전에 정규화한다.
- `metadata.dartStatus`를 통해 데이터 수집 성공 여부를 기록한다.
- 수집된 데이터가 실제 보고서 파이프라인에 정상 반영되는지 끝까지 확인한다.

## 검증 방법
- `/api/data/dart?corp_code=00126380` 호출로 기본 정보 확인
- 삼성전자 등 주요 기업 검색 테스트
- 존재하지 않는 기업명 검색 시의 예외 처리 확인
- `metadata.dartStatus` 값 검증

## 흔한 실수
- 기업명 매칭이 정확하지 않아 엉뚱한 회사의 데이터를 가져오는 경우
- 수치 데이터 파싱 시 콤마(,)나 단위 처리를 잘못하여 값이 왜곡되는 경우
- API 할당량 초과 시에 대한 폴백 처리가 없는 경우
